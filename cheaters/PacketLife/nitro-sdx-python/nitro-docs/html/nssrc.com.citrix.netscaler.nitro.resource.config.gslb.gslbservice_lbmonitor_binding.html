class gslbservice_lbmonitor_binding¶

Bases: nssrc.com.citrix.netscaler.nitro.resource.base.base_resource.base_resource

Binding class showing the lbmonitor that can be bound to gslbservice.

weight¶

Weight to assign to the monitor-service binding. A larger number specifies a greater weight. Contributes to the monitoring threshold, which determines the state of the service.<br/>Minimum value = 1<br/>Maximum value = 100.

servicename¶

Name of the GSLB service.<br/>Minimum length = 1.

monitor_name¶

Monitor name.

monstate¶

State of the monitor bound to gslb service.<br/>Possible values = ENABLED, DISABLED.

monstatcode¶

The code indicating the monitor response.

responsetime¶

Response time of this monitor.

totalfailedprobes¶

The total number of failed probs.

monstatparam2¶

Second parameter for use with message code.

failedprobes¶

Number of the current failed monitoring probes.

monstatparam3¶

Third parameter for use with message code.

monitor_state¶

The running state of the monitor on this service.<br/>Possible values = UP, DOWN, UNKNOWN, BUSY, OUT OF SERVICE, GOING OUT OF SERVICE, DOWN WHEN GOING OUT OF SERVICE, NS_EMPTY_STR, Unknown, DISABLED.

monitortotalprobes¶

Total number of probes sent to monitor this service.

monstatparam1¶

First parameter for use with message code.

monitortotalfailedprobes¶

Total number of failed probes.

monitorcurrentfailedprobes¶

Total number of currently failed probes.
