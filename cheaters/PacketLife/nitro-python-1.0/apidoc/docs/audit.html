Audit

audit configuration.

Configuration APIs

addauditmessageaction

Use this method to adds an audit message action.
The action specifies whether to log the message, and to which log.

addauditnslogaction

Use this method to adds an nslog action.
The action contains a reference to an nslog server and specifies which information to log and how to log that information.

addauditnslogpolicy

Use this method to adds a policy that defines which messages to log to the specified nslog server.

addauditsyslogaction

Use this method to adds a syslog action.
The action contains a reference to a syslog server, and specifies which information to log and how to log that information.

addauditsyslogpolicy

Use this method to adds a policy that defines which messages to log to the specified syslog server.

getauditmessageaction

Use this method to get the current configuration of the specified audit message action.

getauditmessages

Use this method to get the most recent audit log messages.

getauditnslogaction

Use this method to get the current configuration of the specified nslog action.

getauditnslogparams

Use this method to get the current nslog parameter settings.

getauditnslogpolicy

Use this method to get the current configuration of the specified nslog policy.

getauditsyslogaction

Use this method to get the current configuration of the specified syslog action.

getauditsyslogparams

Use this method to get the current syslog parameter settings.

getauditsyslogpolicy

Use this method to get the current configuration of the specified syslog policy.

rmauditmessageaction

Use this method to removes the specified audit message action and associated configuration.

Note: An nslog action cannot be removed if it is bound to an nslog policy.

rmauditnslogpolicy

Use this method to removes the specified nslog policy and associated configuration.

setauditmessageaction_bypasssafetycheck

Use this method to set bypass the safety check and allow unsafe expressions.

setauditmessageaction_loglevel

Use this method to set audit log level, which specifies the severity level of the log message being generated.
The following loglevels are valid:
* EMERGENCY - Events that indicate an immediate crisis on the server.
* ALERT - Events that might require action.
* CRITICAL - Events that indicate an imminent server crisis.
* ERROR - Events that indicate some type of error.
* WARNING - Events that require action in the near future.
* NOTICE - Events that the administrator should know about.
* INFORMATIONAL - All but low-level events.
* DEBUG - All events, in extreme detail.

setauditmessageaction_logtonewnslog

Use this method to set send the message to the new nslog.

setauditmessageaction_stringbuilderexpr

Use this method to set default-syntax expression that defines the format and content of the log message.

setauditnslogaction_acl

Use this method to set log access control list (ACL) messages.

setauditnslogaction_appflowexport

Use this method to set export log messages to AppFlow collectors.
Appflow collectors are entities to which log messages can be sent so that some action can be performed on them.

setauditnslogaction_dateformat

Use this method to set format of dates in the logs.
Supported formats are:
* MMDDYYYY - U.S. style month/date/year format.
* DDMMYYYY - European style date/month/year format.
* YYYYMMDD - ISO style year/month/date format.

setauditnslogaction_logfacility

Use this method to set facility value, as defined in RFC 3164, assigned to the log message.
Log facility values are numbers 0 to 7 (LOCAL0 through LOCAL7). Each number indicates where a specific message originated from, such as the NetScaler appliance itself, the VPN, or external.

setauditnslogaction_loglevel

Use this method to set audit log level, which specifies the types of events to log.
Available settings function as follows:

* NONE - No events.

setauditnslogaction_timezone

Use this method to set time zone used for date and timestamps in the logs.
Available settings function as follows:
* GMT_TIME. Coordinated Universal Time.
* LOCAL_TIME. The server’s timezone setting.

setauditnslogaction_userdefinedauditlog

Use this method to set log user-configurable log messages to nslog.
Setting this parameter to NO causes auditing to ignore all user-configured message actions. Setting this parameter to YES causes auditing to log user-configured message actions that meet the other logging criteria.
