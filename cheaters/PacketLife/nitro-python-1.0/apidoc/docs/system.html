System

system configuration.

Configuration APIs

addsystemcmdpolicy

Use this method to adds a method policy to the system. A method policy specifies the access rights of the system user. By default, the appliance already has the following policies defined:
* operator
* read-only
* network
* superuser

addsystemgroup

Use this method to creates a system-user group, to which you can bind individual users by using the bind system group method.

addsystemuser

Use this method to adds a new user to the system.
Note: You must provide the password after the user name.

bindsystemglobal_policy

Use this method to bind policy to system global.

bindsystemgroup_policy

Use this method to bind policy to system group.

bindsystemgroup_user

Use this method to bind user to system group.

bindsystemuser_policy

Use this method to bind policy to system user.

createsystembackup

Use this method to creates a backup file (*.tgz) that is stored in the /var/ns_sys_backup/ directory. This file can be used to restore the appliance by using the "restore system backup" method.

getsystembackup

Use this method to get the backed up files that were created in the appliance.

getsystemcmdpolicy

Use this method to get information about all configured system method policies, or about the specified policy.

getsystemcollectionparam

Use this method to get collection parameters for historical charting present in nscollect.ini file.

getsystemcore

Use this method to get entities in historical data.

getsystemcountergroup

Use this method to get available counter groups.

getsystemcounters

Use this method to get entities in historical data.

getsystemdatasource

Use this method to get entities in historical data.

getsystementity

Use this method to get entities in historical data.

getsystementitydata

Use this method to get the historical data for entity specific counters.

getsystementitytype

Use this method to get available entity types.

getsystemeventhistory

Use this method to get events in historical data.

getsystemglobal

Use this method to get information about all global policy bindings.

getsystemglobaldata

Use this method to get historical data for global counters.

getsystemgroup

Use this method to get information about all system groups configured on the appliance, or about the specified group.

getsystemparameter

Use this method to get information about the system parameters.

getsystemsession

Use this method to get information about all current system sessions, or about the specified session. The system might reclaim sessions with no active connections before expiry time.

getsystemuser

Use this method to get information about all system users configured on the appliance, or about the specified user.

killsystemsession

Use this method to kills one system session, or all system sessions except the current session.

restoresystembackup

Use this method to restores an appliance by using the backup file (*.tgz) that was created by using the "create system backup" method.

rmsystembackup

Use this method to removes a backup file (*.tgz) that was created by using the "create system backup" method.

rmsystemcmdpolicy

Use this method to removes a method policy from the appliance.
Note: You cannot remove method policies that are bound to a system user.

rmsystementitydata

Use this method to removes the specified entity from historical charting along with all the associated counters till the current time stamp.

rmsystemgroup

Use this method to removes a system group from the appliance.

rmsystemuser

Use this method to removes a system user from the appliance.

setsystemcmdpolicy_action

Use this method to set action to perform when a request matches the policy.
This is mandatory parameter.

setsystemcmdpolicy_cmdspec

Use this method to set regular expression specifying the data that matches the policy.
This is mandatory parameter.

setsystemcollectionparam_community

Use this method to set sNMPv1 community name for authentication.
NOTE: This attribute is deprecated.
This is deprecated argument.

setsystemcollectionparam_datapath

Use this method to set specify the data path to the database.

setsystemcollectionparam_loglevel

Use this method to set specify the log level. Possible values CRITICAL,WARNING,INFO,DEBUG1,DEBUG2

setsystemgroup_promptstring

Use this method to set string to display at the method-line prompt. Can consist of letters, numbers, hyphen (-), period (.), hash (#), space ( ), at (@), equal (=), colon (:), underscore (_), and the following variables:
* %u - Will be replaced by the user name.
* %h - Will be replaced by the hostname of the NetScaler appliance.
* %t - Will be replaced by the current time in 12-hour format.
* %T - Will be replaced by the current time in 24-hour format.
* %d - Will be replaced by the current date.
* %s - Will be replaced by the state of the NetScaler appliance.

Note: The 63-character limit for the length of the string does not apply to the characters that replace the variables.

setsystemgroup_timeout

Use this method to set cLI session inactivity timeout, in seconds.If Restrictedtimeout argument of system parameter is enabled, Timeout can have values in the range [300-86400] seconds. If Restrictedtimeout argument of system parameter is disabled, Timeout can have values in the range [0, 10-100000000] seconds. Default value is 900 seconds.

setsystemparameter_localauth

Use this method to set when enabled, local users can access NetScaler even when external authentication is configured. When disabled, local users are not allowed to access the NetScaler, Local users can access the NetScaler only when the configured external authentication servers are unavailable.

setsystemparameter_natpcbforceflushlimit

Use this method to set flush the system if the number of Network Address Translation Protocol Control Blocks (NATPCBs) exceeds this value.

setsystemparameter_natpcbrstontimeout

Use this method to set send a reset signal to client and server connections when their NATPCBs time out. Avoids the buildup of idle TCP connections on both the sides.

setsystemparameter_rbaonresponse

Use this method to set enable or disable Role-Based Authentication (RBA) on responses.

setsystemparameter_restrictedtimeout

Use this method to set enable/Disable the restricted timeout behaviour. When enabled, timeout cannot be configured beyond admin configured timeout and also it will have\
the [minimum - maximum] range check. When disabled, timeout will have the old behaviour. By default the value is disabled

If Restrictedtimeout argument of system parameter is disabled, Timeout can have values in the range [0, 10-100000000] seconds. Default value is 900 seconds.

setsystemuser_externalauth

setsystemuser_password
