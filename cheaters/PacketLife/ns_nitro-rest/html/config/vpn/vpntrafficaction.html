vpntrafficaction

Configuration for VPN traffic action resource.

Read/write properties

name<String>

Name for the traffic action. Must begin with an ASCII alphabetic or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters. Cannot be changed after a traffic action is created. The following requirement applies only to the NetScaler CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, "my action" or 'my action').
Minimum length = 1

qual<String>

Protocol, either HTTP or TCP, to be used with the action.
Possible values = http, tcp

apptimeout<Double>

Maximum amount of time, in minutes, a user can stay logged on to the web application.
Minimum value = 1
Maximum value = 715827

sso<String>

Provide single sign-on to the web application.
Possible values = ON, OFF

hdx<String>

Provide hdx proxy to the ICA traffic.
Possible values = ON, OFF

formssoaction<String>

Name of the form-based single sign-on profile. Form-based single sign-on allows users to log on one time to all protected applications in your network, instead of requiring them to log on separately to access each one.

fta<String>

Specify file type association, which is a list of file extensions that users are allowed to open.
Possible values = ON, OFF

wanscaler<String>

Use the Repeater Plug-in to optimize network traffic.
Possible values = ON, OFF

kcdaccount<String>

Kerberos constrained delegation account name.
Default value: "Default"
Minimum length = 1
Maximum length = 32

samlssoprofile<String>

Profile to be used for doing SAML SSO to remote relying party.
Minimum length = 1

proxy<String>

IP address and Port of the proxy server to be used for HTTP access for this request.
Minimum length = 1

userexpression<String>

expression that will be evaluated to obtain username for SingleSignOn.
Maximum length = 256

passwdexpression<String>

expression that will be evaluated to obtain password for SingleSignOn.
Maximum length = 256

Read only properties

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
