auditmessages

Configuration for audit message resource.

Read/write properties

loglevel<String[]>

Audit log level filter, which specifies the types of events to display. The following loglevels are valid: * ALL - All events. * EMERGENCY - Events that indicate an immediate crisis on the server. * ALERT - Events that might require action. * CRITICAL - Events that indicate an imminent server crisis. * ERROR - Events that indicate some type of error. * WARNING - Events that require action in the near future. * NOTICE - Events that the administrator should know about. * INFORMATIONAL - All but low-level events. * DEBUG - All events, in extreme detail.
Possible values = ALL, EMERGENCY, ALERT, CRITICAL, ERROR, WARNING, NOTICE, INFORMATIONAL, DEBUG

numofmesgs<Double>

Number of log messages to be displayed.
Default value: 20
Minimum value = 1
Maximum value = 256

Read only properties

value<String>

The Audit message.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK

Query-parameters:

args

http://<NSIP>/nitro/v1/config/auditmessages?args=
      "loglevel":<String[]_value>,
      "numofmesgs":<Double_value>,

Use this query-parameter to get auditmessages resources based on additional properties.

filter

http://<NSIP>/nitro/v1/config/auditmessages?filter=property-name1:property-val1,property-name2:property-val2

Use this query-parameter to get the filtered set of auditmessages resources configured on NetScaler.Filtering can be done on any of the properties of the resource.

view

http://<NS_IP>/nitro/v1/config/auditmessages?view=summary

Use this query-parameter to get the summary output of auditmessages resources configured on NetScaler.

pagesize=#no&pageno=#no

http://<NS_IP>/nitro/v1/config/auditmessages?pagesize=#no&pageno=#no

Use this query-parameter to get the auditmessages resources in chunks.

warning

http://<NS_IP>/nitro/v1/config/auditmessages?warning=yes

Use this query parameter to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning. Other possible values for severity are ERROR, INFO and NONE.
