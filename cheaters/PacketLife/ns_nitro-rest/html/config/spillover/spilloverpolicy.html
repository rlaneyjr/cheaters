spilloverpolicy

Configuration for Spillover policy resource.

Read/write properties

name<String>

Name of the spillover policy.

rule<String>

Expression to be used by the spillover policy.

action<String>

Action for the spillover policy. Action is created using add spillover action command.

comment<String>

Any comments that you might want to associate with the spillover policy.

newname<String>

New name for the spillover policy. Must begin with an ASCII alphabetic or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters. Choose a name that reflects the function that the policy performs. The following requirement applies only to the NetScaler CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, "my policy" or 'my policy').
Minimum length = 1

Read only properties

hits<Double>

The number of times the policy has been hit.

undefhits<Double>

Number of policy UNDEF hits.

builtin<String[]>

Flag to determine if compression policy is builtin or not.
Possible values = MODIFIABLE, DELETABLE, IMMUTABLE

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
