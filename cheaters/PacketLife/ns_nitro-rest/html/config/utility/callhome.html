callhome

Configuration for callhome resource.

Read/write properties

emailaddress<String>

The contact person's E-mail address.
Maximum length = 78

proxymode<String>

Deploy the callhome proxy mode.
Default value: NO
Possible values = YES, NO

ipaddress<String>

Proxy Server IP address.
Minimum length = 1

port<Integer>

Proxy Server Port.
Minimum value = 1
Range 1 - 65535

Read only properties

sslcardfirstfailure<String>

First occurrence SSL card failure.

sslcardlatestfailure<String>

Latest occurrence SSL card failure.

powfirstfail<String>

First occurrence power supply unit failure.

powlatestfailure<String>

Latest occurrence power supply unit failure.

hddfirstfail<String>

First occurrence hard disk drive failure.

hddlatestfailure<String>

Latest occurrence hard disk drive failure.

flashfirstfail<String>

First occurrence compact flash failure.

flashlatestfailure<String>

Latest occurrence compact flush failure.

restartlatestfail<String>

Latest occurrence warm restart failure.

callhomestatus<String[]>

Callhome feature enabled/disable, register with upload server successful/failed.
Possible values = ENABLED, DISABLED, SUCCESSFUL, FAILED

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
