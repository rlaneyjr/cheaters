ipsecprofile

Configuration for IPSEC profile resource.

Read/write properties

name<String>

The name of the ipsec profile.
Minimum length = 1
Maximum length = 32

ikeversion<String>

IKE Protocol Version.
Possible values = V1, V2

encalgo<String[]>

Type of encryption algorithm.
Possible values = AES, 3DES

hashalgo<String[]>

Type of hashing algorithm.
Possible values = HMAC_SHA1, HMAC_SHA256, HMAC_SHA384, HMAC_SHA512, HMAC_MD5

lifetime<Double>

Lifetime of IKE SA in seconds. Lifetime of IPSec SA will be (lifetime of IKE SA/8).
Minimum value = 480
Maximum value = 31536000

psk<String>

Pre shared key value.

publickey<String>

Public key file path.

privatekey<String>

Private key file path.

peerpublickey<String>

Peer public key file path.

livenesscheckinterval<Double>

Number of seconds after which a notify payload is sent to check the liveliness of the peer. Additional retries are done as per retransmit interval setting. Zero value disables liveliness checks.
Minimum value = 0
Maximum value = 64999

replaywindowsize<Double>

IPSec Replay window size for the data traffic.
Minimum value = 0
Maximum value = 16384

ikeretryinterval<Double>

IKE retry interval for bringing up the connection.
Minimum value = 60
Maximum value = 3600

retransmissiontime<Double>

The interval in seconds to retry sending the IKE messages to peer, three consecutive attempts are done with doubled interval after every failure.
Minimum value = 1
Maximum value = 99

Read only properties

builtin<String[]>

Indicates that a variable is a built-in (SYSTEM INTERNAL) type.
Possible values = MODIFIABLE, DELETABLE, IMMUTABLE

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
