systemeventhistory

Configuration for event history resource.

Read/write properties

starttime<String>

Specify start time in mmddyyyyhhmm to start collecting values from that timestamp.

endtime<String>

Specify end time in mmddyyyyhhmm upto which values have to be collected.

last<Integer>

Last is literal way of saying a certain time period from the current moment. Example: -last 1 hour, -last 1 day, et cetera.
Default value: 1

unit<String>

Specify the time period from current moment. Example 1 x where x = hours/ days/ years.
Possible values = HOURS, DAYS, MONTHS

datasource<String>

Specifies the source which contains all the stored counter values.

Read only properties

response<String>

.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
