pqbinding

Configuration for PQ bindings resource.

Read/write properties

vservername<String>

Name of the load balancing virtual server for which to display the priority queuing policy.
Minimum length = 1

Read only properties

policyname<String>

The name of the priority queuing policy.

rule<String>

The condition for applying the policy.

priority<Double>

The priority of queuing the request.

weight<Double>

Weight.

qdepth<Double>

Queue Depth.

polqdepth<Double>

Policy Queue Depth.

hits<Double>

Total number of hits.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK

Use this query-parameter to get the filtered set of pqbinding resources configured on NetScaler.Filtering can be done on any of the properties of the resource.

view

http://<NS_IP>/nitro/v1/config/pqbinding?view=summary

Use this query-parameter to get the summary output of pqbinding resources configured on NetScaler.

pagesize=#no&pageno=#no

http://<NS_IP>/nitro/v1/config/pqbinding?pagesize=#no&pageno=#no

Use this query-parameter to get the pqbinding resources in chunks.

warning

http://<NS_IP>/nitro/v1/config/pqbinding?warning=yes

Use this query parameter to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning. Other possible values for severity are ERROR, INFO and NONE.
