autoscalepolicy

Configuration for Autoscale policy resource.

Read/write properties

name<String>

The name of the autoscale policy.
Minimum length = 1

rule<String>

The rule associated with the policy.

action<String>

The autoscale profile associated with the policy.
Minimum length = 1

comment<String>

Comments associated with this autoscale policy.

logaction<String>

The log action associated with the autoscale policy.

newname<String>

The new name of the autoscale policy.
Minimum length = 1

Read only properties

hits<Double>

Number of hits.

undefhits<Double>

Number of Undef hits.

priority<Double>

Specifies the priority of the policy.

activepolicy<Integer>

Indicates whether policy is bound or not.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
