autoscaleaction

Configuration for autoscale action resource.

Read/write properties

name<String>

ActionScale action name.
Minimum length = 1

type<String>

The type of action.
Possible values = SCALE_UP, SCALE_DOWN

profilename<String>

AutoScale profile name.
Minimum length = 1

parameters<String>

Parameters to use in the action.
Minimum length = 1

vmdestroygraceperiod<Double>

Time in minutes a VM is kept in inactive state before destroying.
Default value: 10

quiettime<Double>

Time in seconds no other policy is evaluated or action is taken.
Default value: 300

vserver<String>

Name of the vserver on which autoscale action has to be taken.

Read only properties

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
