nsevents

Configuration for events resource.

Read/write properties

eventno<Double>

Event number starting from which events must be shown.

Read only properties

time<Double>

Event no.

eventcode<Double>

event Code.

devid<Double>

Device Name.

devname<String>

Device Name.

text<String>

Event no.

additional event information.

data1<Double>

additional event information.

data2<Double>

additional event information.

data3<Double>

additional event information.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK

get (all)count

get (all)

URL:http://<NSIP>/nitro/v1/config/nsevents

Query-parameters:

args

http://<NSIP>/nitro/v1/config/nsevents?args=
      "eventno":<Double_value>,

Use this query-parameter to get nsevents resources based on additional properties.

filter

http://<NSIP>/nitro/v1/config/nsevents?filter=property-name1:property-val1,property-name2:property-val2

Use this query-parameter to get the filtered set of nsevents resources configured on NetScaler.Filtering can be done on any of the properties of the resource.

view

http://<NS_IP>/nitro/v1/config/nsevents?view=summary

Use this query-parameter to get the summary output of nsevents resources configured on NetScaler.

pagesize=#no&pageno=#no

http://<NS_IP>/nitro/v1/config/nsevents?pagesize=#no&pageno=#no

Use this query-parameter to get the nsevents resources in chunks.

warning

http://<NS_IP>/nitro/v1/config/nsevents?warning=yes

Use this query parameter to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning. Other possible values for severity are ERROR, INFO and NONE.
