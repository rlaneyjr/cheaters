nssimpleacl

Configuration for simple ACL resource.

Read/write properties

aclname<String>

Name for the simple ACL rule. Must begin with an ASCII alphabetic or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters. Cannot be changed after the simple ACL rule is created.
Minimum length = 1

aclaction<String>

Drop incoming IPv4 packets that match the simple ACL rule.
Possible values = DENY

td<Double>

Integer value that uniquely identifies the traffic domain in which you want to configure the entity. If you do not specify an ID, the entity becomes part of the default traffic domain, which has an ID of 0.
Minimum value = 0
Maximum value = 4094

srcip<String>

IP address to match against the source IP address of an incoming IPv4 packet.

destport<Integer>

Port number to match against the destination port number of an incoming IPv4 packet. Omitting the port number creates an all-ports simple ACL rule, which matches any port. In that case, you cannot create another simple ACL rule specifying a specific port and the same source IPv4 address.

protocol<String>

Protocol to match against the protocol of an incoming IPv4 packet. You must set this parameter if you have set the Destination Port parameter.
Possible values = TCP, UDP

ttl<Double>

Number of seconds, in multiples of four, after which the simple ACL rule expires. If you do not want the simple ACL rule to expire, do not specify a TTL value.
Minimum value = 4
Maximum value = 2147483647

estsessions<Boolean>

.

Read only properties

hits<Double>

Number of hits for this ACL rule.

__count<Double>

count parameter

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
