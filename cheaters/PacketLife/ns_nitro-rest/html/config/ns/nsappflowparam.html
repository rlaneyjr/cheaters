nsappflowparam

Configuration for appflowParam resource.

Read/write properties

templaterefresh<Double>

IPFIX template refresh interval (in seconds).
Default value: 600
Minimum value = 60
Maximum value = 3600

udppmtu<Double>

MTU to be used for IPFIX UDP packets.
Default value: 1472
Minimum value = 128
Maximum value = 1472

httpurl<String>

Enable AppFlow HTTP URL logging.
Default value: DISABLED
Possible values = ENABLED, DISABLED

httpcookie<String>

Enable AppFlow HTTP cookie logging.
Default value: DISABLED
Possible values = ENABLED, DISABLED

httpreferer<String>

Enable AppFlow HTTP referer logging.
Default value: DISABLED
Possible values = ENABLED, DISABLED

httpmethod<String>

Enable AppFlow HTTP method logging.
Default value: DISABLED
Possible values = ENABLED, DISABLED

httphost<String>

Enable AppFlow HTTP host logging.
Default value: DISABLED
Possible values = ENABLED, DISABLED

httpuseragent<String>

Enable AppFlow HTTP user-agent logging.
Default value: DISABLED
Possible values = ENABLED, DISABLED

clienttrafficonly<String>

Control whether AppFlow records should be generated only for client-side traffic.
Default value: NO
Possible values = YES, NO

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
