transformprofile

Configuration for URL Transformation profile resource.

Read/write properties

name<String>

Name for the URL transformation profile. Must begin with a letter, number, or the underscore character (_), and must contain only letters, numbers, and the hyphen (-), period (.) pound (#), space ( ), at (@), equals (=), colon (:), and underscore characters. Cannot be changed after the URL transformation profile is added. The following requirement applies only to the NetScaler CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, my transform profile or my transform profile).
Minimum length = 1

type<String>

Type of transformation. Always URL for URL Transformation profiles.
Possible values = URL

onlytransformabsurlinbody<String>

In the HTTP body, transform only absolute URLs. Relative URLs are ignored.
Possible values = ON, OFF

comment<String>

Any comments to preserve information about this URL Transformation profile.

Read only properties

regexforfindingurlinjavascript<String>

Patclass having regexes to find the URLs in JavaScript.

regexforfindingurlincss<String>

Patclass having regexes to find the URLs in CSS.

regexforfindingurlinxcomponent<String>

Patclass having regexes to find the URLs in X-Component.

regexforfindingurlinxml<String>

Patclass having regexes to find the URLs in XML.

additionalreqheaderslist<String>

Patclass having a list of additional request header names that should transformed.

additionalrespheaderslist<String>

Patclass having a list of additional response header names that should transformed.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
