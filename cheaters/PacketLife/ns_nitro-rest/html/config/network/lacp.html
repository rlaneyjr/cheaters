lacp

Configuration for Link aggregation control protocol resource.

Read/write properties

syspriority<Double>

Priority number that determines which peer device of an LACP LA channel can have control over the LA channel. This parameter is globally applied to all LACP channels on the NetScaler appliance. The lower the number, the higher the priority.
Default value: 32768
Minimum value = 1
Maximum value = 65535

ownernode<Double>

The owner node in a cluster for which we want to set the lacp priority. Owner node can vary from 0 to 31. Ownernode value of 254 is used for Cluster.
Default value: 255

Read only properties

devicename<String>

Name of the channel.

mac<String>

LACP system MAC.

flags<Double>

Flags of this channel.

lacpkey<Double>

LACP key of this channel.

clustersyspriority<Double>

LACP system (Cluster) priority.

clustermac<String>

LACP system (Cluster) mac.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
