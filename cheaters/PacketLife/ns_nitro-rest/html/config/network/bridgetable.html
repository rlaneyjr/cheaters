bridgetable

Configuration for bridge table entry resource.

Read/write properties

bridgeage<Double>

Time-out value for the bridge table entries, in seconds. The new value applies only to the entries that are dynamically learned after the new value is set. Previously existing bridge table entries expire after the previously configured time-out value.
Default value: 300
Minimum value = 60
Maximum value = 300

vlan<Double>

VLAN whose entries are to be removed.
Minimum value = 1
Maximum value = 4094

ifnum<String>

INTERFACE whose entries are to be removed.

vxlan<Double>

VXLAN whose entries are to be removed.
Minimum value = 1
Maximum value = 16777215

Read only properties

mac<String>

The MAC address of the target.

vtep<String>

The IP address of the VTEP.

flags<Double>

Display flags,.

channel<Double>

The Tunnel through which bridge entry is learned.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
