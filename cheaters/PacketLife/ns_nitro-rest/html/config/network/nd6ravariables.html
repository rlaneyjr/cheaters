nd6ravariables

Configuration for nd6 Router Advertisment configuration variables resource.

Read/write properties

vlan<Double>

The VLAN number.
Minimum value = 0
Maximum value = 4094

ceaserouteradv<String>

Cease router advertisements on this vlan.
Default value: NO
Possible values = YES, NO

sendrouteradv<String>

whether the router sends periodic RAs and responds to Router Solicitations.
Default value: NO
Possible values = YES, NO

srclinklayeraddroption<String>

Include source link layer address option in RA messages.
Default value: YES
Possible values = YES, NO

onlyunicastrtadvresponse<String>

Send only Unicast Router Advertisements in respond to Router Solicitations.
Default value: NO
Possible values = YES, NO

managedaddrconfig<String>

Value to be placed in the Managed address configuration flag field.
Default value: NO
Possible values = YES, NO

otheraddrconfig<String>

Value to be placed in the Other configuration flag field.
Default value: NO
Possible values = YES, NO

currhoplimit<Double>

Current Hop limit.
Default value: 64
Minimum value = 0
Maximum value = 255

maxrtadvinterval<Double>

Maximum time allowed between unsolicited multicast RAs, in seconds.
Default value: 600
Minimum value = 4
Maximum value = 1800

minrtadvinterval<Double>

Minimum time interval between RA messages, in seconds.
Default value: 198
Minimum value = 3
Maximum value = 1350

linkmtu<Double>

The Link MTU.
Default value: 0
Minimum value = 0
Maximum value = 1500

reachabletime<Double>

Reachable time, in milliseconds.
Default value: 0
Minimum value = 0
Maximum value = 3600000

retranstime<Double>

Retransmission time, in milliseconds.
Default value: 0

defaultlifetime<Integer>

Default life time, in seconds.
Default value: 1800
Minimum value = 0
Maximum value = 9000

Read only properties

lastrtadvtime<Double>

Last RA sent timestamp.

nextrtadvdelay<Double>

Next RA delay.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
