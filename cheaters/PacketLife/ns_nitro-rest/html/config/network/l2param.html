l2param

Configuration for Layer 2 related parameter resource.

Read/write properties

mbfpeermacupdate<Double>

When mbf_instant_learning is enabled, learn any changes in peer's MAC after this time interval, which is in 10ms ticks.
Default value: 10

maxbridgecollision<Double>

Maximum bridge collision for loop detection .
Default value: 20

bdggrpproxyarp<String>

Set/reset proxy ARP in bridge group deployment.
Default value: ENABLED
Possible values = ENABLED, DISABLED

bdgsetting<String>

Bridging settings for C2C behavior.
Default value: DISABLED
Possible values = ENABLED, DISABLED

garponvridintf<String>

Send GARP messagess on VRID-configured interfaces upon failover .
Default value: ENABLED
Possible values = ENABLED, DISABLED

macmodefwdmypkt<String>

MAC mode vserver forward packets destined to VIPs.
Default value: DISABLED
Possible values = ENABLED, DISABLED

usemymac<String>

Set/reset cfg_use_my_mac .
Default value: DISABLED
Possible values = ENABLED, DISABLED

proxyarp<String>

Set/reset cfg_proxy_arp_dr .
Default value: ENABLED
Possible values = ENABLED, DISABLED

garpreply<String>

Set/reset REPLY form of GARP .
Default value: DISABLED
Possible values = ENABLED, DISABLED

mbfinstlearning<String>

Enable instant learning of MAC changes in MBF mode.
Default value: DISABLED
Possible values = ENABLED, DISABLED

rstintfonhafo<String>

Enable the reset interface upon HA failover.
Default value: DISABLED
Possible values = ENABLED, DISABLED

skipproxyingbsdtraffic<String>

Enable the proxying of FreeBSD traffic.
Default value: DISABLED
Possible values = ENABLED, DISABLED

returntoethernetsender<String>

Return to ethernet sender.
Default value: DISABLED
Possible values = ENABLED, DISABLED

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
