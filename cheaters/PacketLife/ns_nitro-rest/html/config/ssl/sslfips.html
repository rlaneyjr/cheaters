sslfips

Configuration for fips resource.

Read/write properties

inithsm<String>

FIPS initialization level. The appliance currently supports Level-2 (FIPS 140-2).
Possible values = Level-2

sopassword<String>

Security officer password that will be in effect after you have configured the HSM.
Minimum length = 1

oldsopassword<String>

Old password for the security officer.
Minimum length = 1

userpassword<String>

The Hardware Security Module's (HSM) User password.
Minimum length = 1

hsmlabel<String>

Label to identify the Hardware Security Module (HSM).
Minimum length = 1

fipsfw<String>

FIPS firmware update.
Possible values = 4.6.1

Read only properties

erasedata<String>

Erase data.
Default value: FIPS_ERASE
Minimum length = 1

serial<Integer>

FIPS card serial number.

majorversion<Integer>

Firmware major version.

minorversion<Integer>

Firmware minor version.

fipshwmajorversion<Integer>

FIPS card hardware major version.

fipshwminorversion<Integer>

FIPS card hardware minor version.

fipshwversionstring<String>

FIPS card hardware extended version string.

flashmemorytotal<Integer>

Total size of the flash memory on card.

flashmemoryfree<Integer>

Total size of free flash memory.

sramtotal<Integer>

Total size of the SRAM memory on card.

sramfree<Integer>

Total size of free SRAM memory.

status<Integer>

Status.

flag<Integer>

Internal Flags.

serialno<String>

FIPS card serial number.

model<String>

FIPS card model info.

state<Integer>

FIPS card state.

firmwarereleasedate<String>

FIPS card firmware revision date.

coresmax<Integer>

Maximum number of crypto cores present in the FIPS card.

coresenabled<Integer>

Number of crypto cores enabled in the FIPS card.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
