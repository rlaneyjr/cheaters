sslservice_sslpolicy_binding

Binding object showing the sslpolicy that can be bound to sslservice.

Read/write properties

priority<Double>

The priority of the policies bound to this SSL service.

gotopriorityexpression<String>

Expression specifying the priority of the next policy which will get evaluated if the current policy rule evaluates to TRUE.

policyname<String>

The SSL policy binding.

servicename<String>

Name of the SSL service for which to set advanced configuration.
Minimum length = 1

labeltype<String>

Type of policy label invocation.
Possible values = vserver, service, policylabel

labelname<String>

Name of the label to invoke if the current policy rule evaluates to TRUE.

invoke<Boolean>

Invoke flag. This attribute is relevant only for ADVANCED policies.

Read only properties

polinherit<Double>

Whether the bound policy is a inherited policy or not.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
