sslfipssimtarget

Configuration for FIPS SIM Target resource.

Read/write properties

keyvector<String>

Name of and, optionally, path to the target FIPS appliance's key vector. /nsconfig/ssl/ is the default path.
Minimum length = 1

sourcesecret<String>

Name of and, optionally, path to the source FIPS appliance's secret data. /nsconfig/ssl/ is the default path.
Minimum length = 1

certfile<String>

Name of and, optionally, path to the source FIPS appliance's certificate file. /nsconfig/ssl/ is the default path.
Minimum length = 1

targetsecret<String>

Name for and, optionally, path to the target FIPS appliance's secret data. The default input path for the secret data is /nsconfig/ssl/.
Minimum length = 1

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
