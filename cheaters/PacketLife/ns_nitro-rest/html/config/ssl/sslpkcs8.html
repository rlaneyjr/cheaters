sslpkcs8

Configuration for pkcs8 resource.

Read/write properties

pkcs8file<String>

Name for and, optionally, path to, the output file where the PKCS#8 format key file is stored. /nsconfig/ssl/ is the default path.
Maximum length = 63

keyfile<String>

Name of and, optionally, path to the input key file to be converted from PEM or DER format to PKCS#8 format. /nsconfig/ssl/ is the default path.
Maximum length = 63

keyform<String>

Format in which the key file is stored on the appliance.
Default value: PEM
Possible values = DER, PEM

password<String>

Password to assign to the file if the key is encrypted. Applies only for PEM format files.
Minimum length = 1
Maximum length = 31

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
