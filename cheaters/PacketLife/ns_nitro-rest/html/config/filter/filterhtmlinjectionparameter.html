filterhtmlinjectionparameter

Configuration for HTML injection parameter resource.

Read/write properties

rate<Double>

For a rate of x, HTML injection is done for 1 out of x policy matches.
Default value: 1
Minimum value = 1

frequency<Double>

For a frequency of x, HTML injection is done at least once per x milliseconds.
Default value: 1
Minimum value = 1

strict<String>

Searching for <html> tag. If this parameter is enabled, HTML injection does not insert the prebody or postbody content unless the <html> tag is found.
Default value: ENABLED
Possible values = ENABLED, DISABLED

htmlsearchlen<Double>

Number of characters, in the HTTP body, in which to search for the <html> tag if strict mode is set.
Default value: 1024
Minimum value = 1

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
