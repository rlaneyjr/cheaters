lbwlm

Configuration for web log manager resource.

Read/write properties

wlmname<String>

The name of the Work Load Manager.
Minimum length = 1

ipaddress<String>

The IP address of the WLM.

port<Integer>

The port of the WLM.
Range 1 - 65535

lbuid<String>

The LBUID for the Load Balancer to communicate to the Work Load Manager.

katimeout<Double>

The idle time period after which NS would probe the WLM. The value ranges from 1 to 1440 minutes.
Default value: 2
Minimum value = 0
Maximum value = 1440

Read only properties

secure<String>

Use this parameter to enable secure mode of communication with WLM.
Possible values = YES, NO

state<String>

State of the WLM.
Possible values = ACTIVE, INACTIVE, UNKNOWN

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
