dnsaction

Configuration for DNS action resource.

Read/write properties

actionname<String>

Name of the dns action.

actiontype<String>

The type of DNS action that is being configured.
Possible values = ViewName, GslbPrefLoc, noop, Drop, Cache_Bypass, Rewrite_Response

ipaddress<String[]>

List of IP address to be returned in case of rewrite_response actiontype. They can be of IPV4 or IPV6 type. In case of set command We will remove all the IP address previously present in the action and will add new once given in set dns action command.

ttl<Double>

Time to live, in seconds.
Default value: 3600
Minimum value = 0
Maximum value = 2147483647

viewname<String>

The view name that must be used for the given action.

preferredloclist<String[]>

The location list in priority order used for the given action.
Minimum length = 1

Read only properties

drop<String>

The dns packet must be dropped.
Possible values = YES, NO

cachebypass<String>

By pass dns cache for this.
Possible values = YES, NO

builtin<String[]>

Flag to determine whether DNS action is default or not.
Possible values = MODIFIABLE, DELETABLE, IMMUTABLE

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
