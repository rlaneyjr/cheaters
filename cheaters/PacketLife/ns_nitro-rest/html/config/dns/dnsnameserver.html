dnsnameserver

Configuration for name server resource.

Read/write properties

ip<String>

IP address of an external name server or, if the Local parameter is set, IP address of a local DNS server (LDNS).
Minimum length = 1

dnsvservername<String>

Name of a DNS virtual server. Overrides any IP address-based name servers configured on the NetScaler appliance.
Minimum length = 1

local<Boolean>

Mark the IP address as one that belongs to a local recursive DNS server on the NetScaler appliance. The appliance recursively resolves queries received on an IP address that is marked as being local. For recursive resolution to work, the global DNS parameter, Recursion, must also be set. If no name server is marked as being local, the appliance functions as a stub resolver and load balances the name servers.

state<String>

Administrative state of the name server.
Default value: ENABLED
Possible values = ENABLED, DISABLED

type<String>

Protocol used by the name server. UDP_TCP is not valid if the name server is a DNS virtual server configured on the appliance.
Default value: UDP
Possible values = UDP, TCP, UDP_TCP

Read only properties

servicename<String>

The name of the dns vserver.

port<Integer>

Port of the service.
Range 1 - 65535

nameserverstate<String>

State of the server.
Possible values = UP, DOWN, UNKNOWN, BUSY, OUT OF SERVICE, GOING OUT OF SERVICE, DOWN WHEN GOING OUT OF SERVICE, NS_EMPTY_STR, Unknown, DISABLED

clmonowner<Double>

Tells the mon owner of the service.

clmonview<Double>

Tells the view id by which state of the service is updated.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
