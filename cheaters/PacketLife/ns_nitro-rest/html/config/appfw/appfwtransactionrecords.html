appfwtransactionrecords

Configuration for Application firewall transaction record resource.

Read only properties

httptransactionid<Double>

The http transaction identifier.

packetengineid<Integer>

The packet engine identifier.

appfwsessionid<String>

The session identifier set by the Application Firewall to track the user session.

profilename<String>

Application Firewall profile name.

url<String>

Request URL.

clientip<String>

The IP address of client.

destip<String>

The IP address of destination.

starttime<String>

Conveys time at which request processing started.

endtime<String>

Conveys time at which request processing end.

requestcontentlength<Integer>

The content length of request.

requestyields<Double>

The number of times yielded during request processing to send heart beat packets.

requestmaxprocessingtime<Double>

The maximum processing time across yields during request processing.

responsecontentlength<Integer>

The content length of response.

responseyields<Double>

The number of times yielded during response processing to send heart beat packets.

responsemaxprocessingtime<Double>

The maximum processing time across yields during response processing.

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
