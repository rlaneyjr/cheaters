appfwconfidfield

Configuration for configured confidential form fields resource.

Read/write properties

fieldname<String>

Name of the form field to designate as confidential.
Minimum length = 1

url<String>

URL of the web page that contains the web form.
Minimum length = 1

isregex<String>

Method of specifying the form field name. Available settings function as follows: * REGEX. Form field is a regular expression. * NOTREGEX. Form field is a literal string.
Default value: NOTREGEX
Possible values = REGEX, NOTREGEX

comment<String>

Any comments to preserve information about the form field designation.

state<String>

Enable or disable the confidential field designation.
Default value: ENABLED
Possible values = ENABLED, DISABLED

Read only properties

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
