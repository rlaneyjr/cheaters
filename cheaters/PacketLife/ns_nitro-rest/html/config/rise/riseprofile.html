riseprofile

Configuration for RISE profile resource.

Read/write properties

profilename<String>

Name of the RISE profile.
Minimum length = 1
Maximum length = 83

Read only properties

servicename<String>

RISE Service name.
Minimum length = 1
Maximum length = 31

deviceid<String>

Device id.
Minimum length = 1
Maximum length = 31

slotid<Double>

Slot number of the RISE profile.

slotno<Double>

Slot number of the RISE profile.

vdcid<Double>

RISE vdc id.

vpcid<Double>

RISE vpc id.

ipaddress<String>

RISE ip address.

mode<String>

RISE attach mode.
Possible values = Direct, Indirect, vPC-Direct

status<String>

RISE status.
Possible values = Active, Inactive

vlan<Double>

RISE Vlan id.

vlangroupid<Double>

RISE Vlan Group id.

vlancfgstatus<String>

RISE config status.
Possible values = Ok, Invalid

ifnum<String>

RISE Interface number.

issu<String>

RISE issu status.
Possible values = InProgress, None

__count<Double>

count parameter

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

warning   <String_value>

Specify to get warnings in nitro response. If this field is set to YES, warning message will be sent in 'message' field and 'WARNING' value is set in severity field of the response in case there is a warning.
Other possible values for severity are ERROR, INFO and NONE.
Default value: NO
Possible values = YES, NO

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE, ROLLBACK
