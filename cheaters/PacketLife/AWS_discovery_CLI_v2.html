Using AWS Application Discovery Service Walkthrough

Before you begin using the AWS Application Discovery Service, you need to verify that you have completed the steps in Setting Up AWS Application Discovery Service. By completing the steps in that topic, you ensure the following:

  • You have an AWS account.
  • Your account has been whitelisted and granted access to Application Discovery Service.
  • You have a key pair named discovery.
  • The IAM policy attached to your user account has access to Application Discovery Service resources, including the Arsenal service.

       Check CLI version AWS Application Discovery Service is available on the version of CLI.

 

Also, you need to download and configure the AWS Command Line Interface (CLI). When prompted, specify your whitelisted AWS credentials and set the default region to us-west-2.

Use the AWS Application Discovery Service using the AWS CLI

After you install the AWS Application Discovery agent or agentless Connector on a server in your data center, youre ready for the next section of the walkthrough. This section shows you how to use the AWS Application Discovery Service using the AWS CLI.

1.     Use the following command to get the status and location of installed agents.

aws discovery describe-agents

It lists AWS agents by ID or lists all agents associated with your user account if you did not specify an agent ID as shown below. Please see options of this CLI. Health field describes the current status of agent. Running means the agent is collecting the data successfully. Unknown means the agent does not collect the data yet. Unhealthy means the agent needs to be re-installed.

 

If you want to check the status of a specific set of agents, run the following commands.

aws discovery describe-agents - -agent-ids <value>

 

2.     Use the following command to instruct specific agents or agentless connectors to start collecting data. At this point, agent start sending data to arsenal endpoints.

start-data-collection-by-agent-ids --agent-ids <value>

It instructs the specified agents to start collecting data.

List-configuration (Server, Process, Connection)

 

3.     Use the following command to list discovered servers 

list-configurations --configuration-type SERVER
Below shows the sample output. For each discovered server by agents or agentless connectors, this command line shows the hostname, OS name and OS version. To have more detailed information about discovered server such as CPU and memory usage, please note configuration id of each server you want to know more and run describe-configuration CLI with server configuration id that will be described later.
 

4.     Use the following command to list discovered processes 

list-configurations --configuration-type PROCESS
Below shows the sample output. This CLI shows process command line, process name, process path, and server hostname each process is running on. 
 
 

5.     Use the following command to list all connections 

list-configurations --configuration-type CONNECTIONS
Below shows the sample output. This CLI shows source server IP address, destination server IP address, destination server port, and connection occurrence between timestamp.
If you want to identify all the connections from a specific source server, please use the following command.
list-configurations -configuration-type CONNECTIONS -filters sourceServer.configid <value> -filters destinationserver.configid <value>
If you want to identify all the connections to a specific destination server, please use the following command.
list-configurations -configuration-type CONNECTIONS filters destinationserver.configid <value>
 
If you want to identify all the connections between a specific source server and destination server, please use the following command.
 
list-configurations -configuration-type CONNECTIONS filters sourceServer.configid <value> -filters destinationserver.configid <value>
 
If you want to identify connections between processes, use the following command.
 
list-configurations -configuration-type CONNECTIONS -filters sourceProcess.name <value> filters destinationProcess.name <value> 
 
 
 

6.     Use describe-configuration command to retrieve a list of attributes for server. For example, to describe more details of discovered server such as CPU/Memory usage, use the following command with server configuration id identified in step 3.

describe-configurations --configuration-ids <server configuration id>
Below shows the sample output. CPU type, average CPU usage, maximum CPU usage, average disk read/write IOPS, max disk read/write IOPS, and memory usage.
 

 

 

 

Tag

7.     Use the following command to tag servers 

create-tags --configuration-ids <value> --tags <value>
You need to put a list of configuration items that you want to tag and associate tags with one or more configuration items. For instance, if you want to attach tag webserver to two discovered nginx web servers in step 3, use the following command with server configuration ids identified in step 3.
 
create-tags --configuration-ids server configuration id server configuration id --tags key=servertype,value=webserver
 
 
8.  Use the following command to list tagged servers in step 8.
describe-tags --filters name=tagKey,values=servertype name=tagValue, values=webserver
You can filter the list using a key -value format. You can separate these items by using logical operators. Please see here for details.
 
9.  Use the following command to delete the association between configuration items and one or more tags
delete-tags --tags key=servertype, value=webserver
 
Export

10.  Use the following command to Export all discovered configuration data to an Amazon S3 bucket. The output includes an export ID which you will use to view the export status.

export-configurations

11.  Use the following command to retrieves the status of a given export process.

describe-export-configurations --export-ids <value>
