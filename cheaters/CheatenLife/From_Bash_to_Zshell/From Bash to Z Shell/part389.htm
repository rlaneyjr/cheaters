Input and Output Within Widgets

Many widget functions need to read input from the user, or output a message. Of course, this is just what the line editor is doing for the shell as a whole, but sometimes a widget function has own its reasons for reading or writing characters. Often both are combined in the same function. For example, it’s common for a function to print a message like Type y to proceed, or n to abort, then read a key that the user types. In this section we’ll discuss the mechanisms that let you do that.
