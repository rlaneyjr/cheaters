Splitting Strings

There is also a flag for splitting a string directly. This allows us to use a separator character of our choosing. Taking the fifth field of the password file instead of the first demonstrates how this might be useful. This is because the fifth field often contains spaces that we want to preserve.

In the following example, the s flag defines the separator character. It takes an argument in much the same way as we saw for some glob qualifiers in the section “More Complicated Qualifiers: String Arguments” in Chapter 9. Also similar to glob qualifiers is the way we can clump several flags together in the same parentheses so the o, p, and s flags appear together as ops. The p flag here allows us to use the \n escape sequence to indicate a newline:

echo ${(ops:\n:)"$(cut -d: -f5 /etc/passwd)"}

Splitting the result of expansions into lines like this is quite a common thing to do so there is another flag, f, which provides a shortcut for it. So, the following is the same as the previous example:

echo ${(of)"$(cut -d: -f5 /etc/passwd)"}

We’ll see the f flag again in the section “States” in Chapter 15. It is often useful when generating completion matches.
