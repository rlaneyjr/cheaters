The Null Device

The file /dev/null exists on all Unix, Linux, and similar systems. (The equivalent on Windows is NUL, with no directory name.) Any output sent to /dev/null is simply thrown away by the operating system. This is very useful for redirecting standard output when you aren’t interested in the results. The following runs myprog with standard output thrown away. That might be useful if you know what the output will be and are not interested in it; the unwanted output might clutter up the output of a script, for example.

myprog >/dev/null

You can also redirect input from /dev/null. If the program reads its input, it will immediately see the end of the file. This can be used to stop a program reading from the terminal. /dev/null is often used with redirection; see the section “Redirection” in this chapter for more on that important topic.
