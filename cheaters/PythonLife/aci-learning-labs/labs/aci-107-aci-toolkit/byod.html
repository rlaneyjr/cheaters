Setting up the environment

This tutorial will walk you through installing the acitoolkit using the sources so you will be able to edit the samples and even the acitoolkit source code if so desired.

Prerequisites

Git

You will be performing a 'git clone' of the code from the Git Repository. There are a number of ways of achieving this. For example, an IDE, such as Eclipse, can do this for you. Otherwise, if you don't already have the command line version, you can download and install it from here:

http://git-scm.com/downloads

Python

The Python website offers two major versions of Python: 3.x.x and 2.x.x, and you will need 2.x.x for this lab. If you already have Python installed, you can check the version by simply running it from a terminal or shell:

C:\Users\Cisco>python
Python 2.7.9 (default, Dec 10 2014, 12:28:03) ...

Otherwise, you can download and install it from here:

https://www.python.org/downloads

Editor

If you'd like to modify the source code, then you'll need a text editor of some sort, and virtually any editor will do. But note that in Python, indentation is used to indicate nesting level, so do not mix tabs with spaces in any given source file.

One popular editor specifically "tuned" for Python is called 'idle' and it provides syntax coloring and code debugging. It is included in the Python 2.7.x install and can be invoked like so:

python C:\Python27\Lib\idlelib\idle.py <source.py>

Where <source.py> is the file you wish to edit.

Get the acitoolkit library and tutorial

Next, you must download the the acitoolkit into a directory of your choice. This is done using the git utility installed above:

  git clone https://github.com/datacenter/acitoolkit.git

Install

Next, cd into the created directory and install the acitoolkit::

  cd acitoolkit
  python setup.py install

Note that when installing on Mac or Linux, you will likely need to run this as root so preface the command with the sudo keyword as follows::

  sudo python setup.py install

Setting login credentials

In the acitoolkit/samples directory there is a credentials.py file that contains the login credentials for the APIC and is shown (post edits) in its entirety below. It contains the following variables: LOGIN, PASSWORD, and IPADDR. These variables should be modified to the settings of your particular APIC.

   LOGIN = 'admin'
   PASSWORD = 'cisco123'
   IPADDR = '10.10.20.57'
   URL = 'https://' + IPADDR

Note: There may be another credentials.py file in the parent acitoolkit directory that is ignored and can be deleted. You should be modifying acitoolkit/samples/credentials.py.
